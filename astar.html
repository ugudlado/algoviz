Context

The Problem

Dijkstra finds optimal paths but explores in all directions equally. Can you guide the search toward the goal to find it faster?

The Approach

Combine actual cost from start (g) with estimated cost to goal (h). Always expand the node with lowest f = g + h. The heuristic guides search direction.

When To Use

Game maps, robot navigation, puzzle solving. Faster than Dijkstra when a good distance heuristic exists. Used in Google Maps, game engines.

Pseudocode
open = {start} g[start] = 0 f[start] = h(start, goal) while open is not empty: curr = node in open with min f if curr == goal: return path open.remove(curr) for neighbor of curr: tentative_g = g[curr] + 1 if tentative_g < g[neighbor]: g[neighbor] = tentative_g f[neighbor] = g + h(n, goal) open.add(neighbor)
Real-world Analogy

Smart GPS with traffic awareness — unlike basic GPS (Dijkstra), A* uses a heuristic estimate of remaining distance to guide the search toward the goal, avoiding unnecessary detours and finding the optimal path faster.
