Context

The Problem

A BST degrades to O(n) when data is inserted in sorted order. How do you guarantee O(log n) always?

The Approach

After every insert or delete, check the balance factor (left height − right height) at each node. If |balance| > 1, rotate to restore balance.

When To Use

When guaranteed O(log n) operations matter more than insert speed. Database engines, memory allocators, any ordered collection needing predictable performance.

Rotation Types
LL Left-Left: right rotation at imbalanced node
RR Right-Right: left rotation at imbalanced node
LR Left-Right: left rotation on child, then right rotation
RL Right-Left: right rotation on child, then left rotation
Pseudocode
insert(node, val): BST insert as usual updateHeight(node) bf = height(left) - height(right) if |bf| > 1: determine rotation type apply rotation return balanced node
Real-world Analogy

Real-world analogy: A bookshelf organizer who keeps books sorted but also ensures the shelf is never lopsided. After every addition or removal, they rebalance by rotating sections left or right so no stack is more than one book taller than its neighbor — keeping search time fast no matter how many books you add.
