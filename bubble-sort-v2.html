Bubble Sort

Sorting Time: O(n²)
Time Complexity

Each pass walks the unsorted portion comparing neighbors. With n elements, total comparisons grow quadratically.

Best Case O(n)
Worst Case O(n²)
Space: O(1)
Space Complexity

Bubble sort operates in-place. It requires only a single constant space allocation to temporarily hold a value during the swap operation, regardless of the input array size.

Scenarios
Standard Start A standard unordered array. Good for seeing average-case performance and normal behavior.
Reverse Ordered Every element must bubble all the way across the array, maximizing both comparisons and swaps.
Already Sorted The algorithm detects this in just one pass and terminates early without swapping.
Trapped Small Value Large elements bubble right quickly, but the smallest element at the end takes N passes to crawl left.
Logic
for each pass: no_swap = true for i from 0 to unsorted: if A[i] > A[i+1]: swap(A[i], A[i+1]) no_swap = false if no_swap: done!
