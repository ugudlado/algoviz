The problem

You have a shelf of books in random order and need to sort them by height. You can only compare two neighboring books at a time and swap if they're out of order.

The approach
for each pass along the shelf: no_swap = true for each neighbor pair: if left book taller: swap them no_swap = false if no_swap: done!
Why O(n²)?

Each pass walks the unsorted portion comparing every neighbor. With n elements: n−1 comparisons first pass, then n−2, and so on.

Total: (n−1) + (n−2) + … + 1 = n(n−1)/2 — grows as O(n²).

Early-termination gives O(n) best case (already sorted). Space: O(1) — in-place.
