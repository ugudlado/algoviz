Context

The Problem

You have a set of GPS coordinates for cities. You want to find the smallest polygon that contains all of them — useful for defining a region's boundary.

The Approach

Sort points by angle from the leftmost point. Add each point to the hull; if it makes a right turn with the last two points, remove the middle one (it's inside the hull).

When To Use

Geographic boundary computation, collision detection in games, image processing, robotics path planning, statistical outlier detection.

Pseudocode
find lowest point P0 sort points by polar angle to P0 push P0, P1 onto stack for i from 2 to n-1: while cross(next-to-top, top, P[i]) ≤ 0: pop stack (discard point) push P[i] onto stack stack = convex hull
Real-world Analogy

Rubber band around nails — imagine nails hammered into a board at each point. Stretch a rubber band around all the nails — the shape it forms is the convex hull. Graham Scan finds this shape by sorting points by angle from the lowest nail, then walking around and removing any nail that causes an inward dent.
