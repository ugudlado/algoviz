Pseudocode
sort requests by floor while requests remain: move in current direction if floor has request: service request if at boundary (0 or max): reverse direction if no more requests ahead: reverse direction
Real-world Analogy

An elevator in a building — instead of jumping to each requested floor in order received, the elevator moves in one direction, stopping at all requested floors along the way, then reverses. This is exactly how real elevators work and how disk heads sweep across platters in the SCAN algorithm.
