Context

The Problem

You need to repeatedly find and remove the smallest (or largest) element from a growing collection — like a task scheduler picking the highest-priority job.

The Approach

Store elements in a complete binary tree where every parent is smaller than its children. Insert at the bottom and bubble up. Extract-min swaps root with last, then sinks down.

When To Use

Priority queues, heap sort, Dijkstra's algorithm, task scheduling, median maintenance.

    Pseudocode
    insert(value): heap.push(value) i = heap.length - 1 while i > 0 and heap[i] < heap[parent(i)]: swap(heap[i], heap[parent(i)]) i = parent(i) extractMin(): min = heap[0] heap[0] = heap.pop() i = 0 while smallest child < heap[i]: swap(heap[i], heap[smallest]) i = smallest return min
    Real-world Analogy

    An emergency room triage queue — patients are not seen in arrival order but by severity. The most critical patient (smallest priority number) is always treated first, just like extract-min returns the root.
