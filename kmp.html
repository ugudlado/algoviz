Context

The Problem

You're searching for a pattern in a 1GB text file. The naive approach re-examines characters on mismatch. Can you avoid redundant work?

The Approach

Precompute a 'failure function' that tells you how far to shift the pattern after a mismatch, using partial matches. Never re-examine text characters.

When To Use

Efficient string search in large texts. Text editors (find/replace), virus scanners, network packet inspection.

Pseudocode
// Build failure table fail[0] = 0; len = 0; i = 1 while i < m: if P[i] == P[len]: fail[i++] = ++len else if len > 0: len = fail[len-1] else: fail[i++] = 0 // Search using failure fn i = 0; j = 0 while i < n: if T[i] == P[j]: i++; j++ if j == m: match at i-j else if T[i] != P[j]: j = j>0 ? fail[j-1] : 0
Real-world Analogy

grep and find-replace in text editors — KMP powers fast file searching. The key insight: when a mismatch occurs, the failure function tells you how far back in the pattern to restart, avoiding backtracking in the text. This reduces worst-case from O(n·m) to O(n+m).
