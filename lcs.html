Context

The Problem

Two versions of a document exist. You want to find what they share — the longest sequence of lines that appear in both, in order (like git diff).

The Approach

Build a table where cell (i,j) = length of LCS of first i chars of A and first j chars of B. Each cell uses the cell above, left, or diagonal.

When To Use

Version control (diff), DNA sequence alignment, plagiarism detection, file comparison tools.

for i from 1 to m: for j from 1 to n: if A[i] == B[j]: dp[i][j] = dp[i-1][j-1] + 1 else: dp[i][j] = max( dp[i-1][j], dp[i][j-1] )
Real-world Analogy

Git diff between file versions — Git uses LCS to find matching lines between two versions of a file. The longest common subsequence tells Git which lines are unchanged; everything else is an insertion or deletion.
