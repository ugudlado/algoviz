Context

The Problem

You have a pile of unsorted records — invoices, names, scores. Sorting them one by one is slow. How do you sort a million records efficiently?

The Approach

Divide the list in half repeatedly until you have single elements (already sorted). Then merge pairs back together in order — merging two sorted lists is fast.

When To Use

When you need guaranteed O(n log n) performance. Preferred for linked lists and external sorting (data too large for RAM).

Pseudocode
mergeSort(arr): if len(arr) <= 1: return arr mid = len(arr) / 2 left = mergeSort(arr[:mid]) right = mergeSort(arr[mid:]) return merge(left, right)   merge(L, R): pick smaller of L[i], R[j] append remaining elements
Real-world Analogy

Sorting a deck of cards — split the deck in half repeatedly until you have single cards, then merge pairs back together in order. Two sorted piles merge easily: just always take the smaller top card.
