Context

The Problem

You're programming a chess AI. Each move branches into many possible opponent responses. How do you find the move that leads to the best outcome?

The Approach

Build a game tree. Maximizing player picks the highest-value branch; minimizing player picks the lowest. Alpha-beta pruning skips branches that can't change the outcome.

When To Use

Two-player zero-sum games: chess, checkers, tic-tac-toe, Go. Any adversarial decision problem where you need to anticipate opponent moves.

Pseudocode
minimax(node, depth, isMax, α, β): if terminal: return score if isMax: val = -∞ for child in children: val = max(val, minimax(child)) α = max(α, val) if β ≤ α: break else: val = +∞ for child in children: val = min(val, minimax(child)) β = min(β, val) if β ≤ α: break return val
Real-world Analogy

Chess grandmasters thinking ahead — each player assumes the opponent will make their best move. You think “if I go here, they’ll go there, then I’ll go...” — exploring the game tree. Alpha-beta pruning is like realizing “no matter what I do in this branch, the opponent already has a better option elsewhere” — so you stop exploring it.
