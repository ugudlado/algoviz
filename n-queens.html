Context

The Problem

Place N chess queens on an N×N board so that no two queens attack each other — no two queens share a row, column, or diagonal.

The Approach

Backtracking: place queens one row at a time. If a placement causes a conflict, backtrack to the previous row and try the next column. Prune entire branches the moment a conflict is detected.

When To Use

Constraint satisfaction problems (CSP): Sudoku, graph coloring, scheduling. Any problem where you build a solution incrementally and can detect invalid states early.

Pseudocode
solve(row): if row == n: record solution for col in 0..n-1: if isValid(board, row, col): board[row] = col solve(row + 1) board[row] = -1 ← backtrack isValid(board, row, col): for r in 0..row-1: if board[r] == col: ✗ if |r-row| == |board[r]-col|: ✗ return true
Real-world Analogy

Real-world analogy: Seating feuding guests at a dinner party. Each guest (queen) cannot be seated next to — or even within sight of — any rival. You place guests one row of tables at a time. The moment you see two rivals facing each other, you move the last guest to a different seat. If you run out of valid seats in that row, you backtrack to the previous row and reseat that guest too. Backtracking lets you explore every valid seating arrangement without trying obviously invalid combinations.
