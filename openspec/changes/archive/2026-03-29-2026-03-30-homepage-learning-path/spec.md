# Homepage Learning Path & Discovery

## Summary
Redesign the homepage to be a learning guide, not just a link directory. Add difficulty levels, prerequisites, estimated time, and curated learning paths so students know where to start and what to learn next.

## Acceptance Criteria
1. Each algorithm card on the homepage shows: difficulty badge (Beginner/Intermediate/Advanced), estimated time ("~10 min"), and a one-line description of what you will learn
2. Three curated "Learning Paths" section added: "CS Fundamentals" (sorting → searching → recursion), "Interview Prep" (arrays → trees → graphs → DP), "Systems Thinking" (hashing → caching → scheduling → compression)
3. Each learning path shows algorithms in order with connecting arrows/lines and a progress indicator concept
4. Algorithm cards show prerequisite chips: e.g. Dijkstra shows "Prereq: BFS" as a small tag
5. Homepage search bar becomes functional — filters algorithm cards by name or category in real time
6. "Start Here" hero CTA links to Bubble Sort as the recommended first algorithm
