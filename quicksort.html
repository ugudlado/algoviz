Context

The Problem

You need to sort data in place without extra memory. Merge sort uses O(n) extra space — can you do better?

The Approach

Pick a pivot element. Rearrange so everything smaller is left, everything larger is right. Recursively sort both halves. The pivot is now in its final position.

When To Use

General-purpose in-place sorting. Fastest in practice for random data despite O(n²) worst case. Used in most standard library sort implementations.

Pseudocode
quickSort(arr, lo, hi): if lo < hi: p = partition(arr, lo, hi) quickSort(arr, lo, p - 1) quickSort(arr, p + 1, hi)   partition(arr, lo, hi): pivot = arr[hi] i = lo - 1 for j = lo to hi - 1: if arr[j] <= pivot: i++; swap(arr[i], arr[j]) swap(arr[i+1], arr[hi])
Real-world Analogy

Organizing students by height — pick one student as the pivot, then send everyone shorter to the left and everyone taller to the right. Repeat in each group until everyone is lined up in order.
