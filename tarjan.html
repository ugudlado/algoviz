Context

The Problem

Find all Strongly Connected Components (SCCs) in a directed graph. An SCC is a maximal set of vertices where every vertex is reachable from every other vertex.

The Approach

Run DFS while tracking each node's discovery time and low-link value (the smallest discovery time reachable via its subtree). When a node's low-link equals its own discovery time, it is an SCC root — pop everything above it on the stack.

When To Use

Compiler dependency analysis, detecting cycles in directed graphs, solving 2-SAT problems, finding circular dependencies in package managers, and decomposing graphs into DAGs.

Why O(V+E)?

Tarjan's makes a single DFS pass: each vertex is visited once (O(V)), and each edge is examined once (O(E)). The stack push/pop operations are amortized O(1) per vertex. Total: O(V+E) — optimal for graph traversal.

Real-world analogy: Imagine a city where every street is one-way. An SCC is a neighborhood where you can drive from any intersection to any other using only those one-way streets — you can always find your way home. Tarjan's algorithm finds all such self-contained neighborhoods in one efficient sweep of the street map.

Pseudocode
function strongConnect(v):
  disc[v] = low[v] = index++
  push v onto stack
  onStack[v] = true

  for each neighbor w of v:
    if w not visited:
      strongConnect(w)        // tree edge
      low[v] = min(low[v], low[w])
    elif w is onStack:
      low[v] = min(low[v], disc[w]) // back edge

  if low[v] == disc[v]:       // v is SCC root
    pop stack until v
    report SCC
